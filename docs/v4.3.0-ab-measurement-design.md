# v4.3.0 A/B Measurement Design — resolving "effect unmeasured"

> Status: **DESIGN, rev2** (not yet run). Purpose: turn v4.3.0's honest "transferred procedures, effect not yet A/B-measured" into a falsifiable measurement, so each discipline is **kept or cut on evidence** — closing fablize's anti-bloat loop.
> Lineage: method mirrors `fivetaku/fablize` (19-run A/B + 26 real sessions; "direction solid, decimals not asserted") and `tmdgusya/prometheus`'s on/off plan.
> rev2: hardened against an internal adversarial review (methodology + anti-bloat lenses). The review's load-bearing fixes — hand-score before you build a classifier, pre-register the CUT threshold, demote M3, replay (not re-run) the Stop hook, make false-positive cost first-class, and tag every durable output with model+effort — are folded in below. (One review claim, "UNTESTABLE isn't shipped," was checked and is **false**: it exists at `qa-prompt.md:142/144/418`, so M4 tests a real addition, not a rename.)

---

## 0. What we measure, and the honesty rules

We measure **procedure transfer** — the effect of a *discipline a harness can enforce* — holding the **model fixed**. NOT model capability (out-of-spec discovery, creative quality, self-driven depth); those don't transfer via a harness and are out of scope by construction.

**Honesty rules (inherited from the references, tightened by review):**
- Report **direction (helps / neutral / hurts) + the raw tally** (e.g. `7/15 → 2/15`). Do **not** report magnitude — no "70% fewer," no "31%→13%." At N≈15 you are entitled to a *sign and a count*, not an effect size. If a magnitude is wanted, show a Wilson 95% CI (one line of code) so its width makes the uncertainty visible; never a bare point estimate.
- **Single model family ⇒ results do not port.** Every durable output (results CSV, CHANGELOG status flip) MUST carry `model` + `effort` inline. The caveat must live in the artifact future-readers trust, not only in this doc.
- **A CUT is asymmetric and dangerous.** A false-KEEP costs prompt tokens; a false-CUT *ships a regression to every user*. Therefore **neutral ≠ CUT**: absence of measured benefit at small N is INCONCLUSIVE, never a removal. CUT requires a **positive signal of harm** (a negative point estimate, or a null-fixture false-positive cost over threshold).
- **The measuring instrument must be lighter than, and independent of, the thing it measures** (see §4 — this is where anti-bloat re-enters and must be blocked).

## 2. Method

- **Paired A/B** (same task, same model+effort, ON vs OFF; randomize order; block by task) — for M1, M1b, M4, M5, M6, M7. **Replay** (offline, not paired) for M2. **Cross-tier two-stage** for M3a; **capability arm** for M3b.
- **Pre-registered KEEP/CUT thresholds — written BEFORE running.** Per discipline: (a) the minimum directional margin that counts as KEEP (e.g. ≥3/15 net on the primary), (b) the null-fixture FP threshold that forces CUT (e.g. ≥2/5 false alarms), (c) the default for everything between: **INCONCLUSIVE**.
- **N.** 12-15 pairs reads *direction only* for M1/M4/M5/M6. M2 wants **hundreds** of replayed turns (cheap). M3a/M3b/M7 are **rare-event**: pre-register a minimum event count per arm (≥8 ceiling/non-obvious events) before reading direction, else report INCONCLUSIVE — a 3/15→1/15 read on an engineered rare event is inside the fixture-construction noise.
- **Model fixed; tagged.** Record exact `model`+`effort` on every row; one family per run; never pool across families.
- **Blind, validated scoring** (§4).

## 3. Corpus (fixtures)

Under `tests/ab-corpus/<discipline>/`: a task prompt + a seeded fault + an oracle. Plus **null fixtures** (no fault) for M1, M2, M4 to measure FP cost.

- **M1/M1b** — render tasks with runtime-only defects (off-canvas chart overflow, overlay covering a game board, clipped SVG, exit-0-with-stack-trace CLI). Null: pure-logic phases where observation is unwarranted.
- **M2** — *not new fixtures*: a large corpus of **already-logged real turns** for replay. Null: turns that genuinely end on a question or a complete deliverable.
- **M4** — unreachable-app fixtures (missing creds / won't boot). Null: **reachable** apps (to catch false-UNTESTABLE).
- **M5** — multi-phase builds with an engineered context trap (a vague request whose naive decomposition triggers a concrete downstream rework event = the oracle).
- **M6** — review fixtures seeded with **oracle-tagged** subtle-but-real bugs (tag fixed by the author, independent of the model's printed confidence).
- **M7** — debug fixtures with a non-obvious cause where the loudest log line is a decoy; oracle = the true-cause site for the diff predicate.

**Instrument independence (critical):** the person who seeds the faults must **not** be the person who writes the scorer, and a **held-out fixture set from a different author** is scored by the same frozen scorer. Report KEEP/CUT separately on in-author vs held-out fixtures. If the direction flips on held-out, the discipline was an artifact of the author's mental model, not a real effect.

## 4. Scoring — hand-first, build only when justified

The anti-bloat trap: building six classifiers + a corpus + a permanent "regression guard" to measure disciplines that are a few lines of prompt is **the accretion this is meant to prevent** (prometheus deleted its own `goals.py` engine the moment a lighter path existed). So:

1. **Default = hand-score 12-15 transcripts** (≈ one focused hour). For any discipline whose shipped footprint is < ~30 lines (M1b, M2-completion, M4), do **not** build a classifier — read the transcripts against the oracle by hand.
2. **Build an offline scorer ONLY** where a corpus is reused across many runs and the oracle is crisp — realistically just **M1's render-leak detector** (does the final verdict match the seeded defect?) and **M2's premature-stop regex** (reuse `hooks/finish-the-work.sh`'s own pattern, run offline).
3. **A scorer that drives a CUT must first be validated:** dry-run it against a hand-labeled subset and report its own false-pos/neg rate. An unvalidated scorer is not a judge. "Pure function" ≠ "correct."
4. **Do not enshrine** unvalidated scorers as permanent release guards. A scorer earns guard status only after step 3.
5. Results row schema (`tests/ab-results/<discipline>.csv`): **`model, effort,`** condition, fixture, in-author|held-out, primary-metric, fp-metric, raw-evidence-pointer. The first two are mandatory (§0).

## 5. Threats to validity

1. **Capability/procedure entanglement** — M3's reason for the split above. Do not report a pooled M3 number.
2. **Instrument non-independence (the dangerous one)** — same author seeds the fault, writes the oracle, AND writes the scorer ⇒ the grader is bias-locked to the corpus, scoring KEEP because author intuition is baked into both. Mitigated by §3's author separation + held-out set. This is *distinct* from corpus-selection bias.
3. **Small N / rare events** — direction only; pre-registered event minimums (§2). Never publish a magnitude.
4. **Single model family** — does not port; tagged in every durable output (§0, §4).
5. **Observer effect / spillover** — toggle one discipline at a time.
6. **Asymmetric harm** — false-CUT ships a regression; false-UNTESTABLE (M4) hides defects behind honesty; false-block (M2) nags correct stops. All three are weighted heavier than their symmetric benefit in §6, not 1:1.
7. **Determinism gap** — M2 is a fixed function; replay, don't re-run.

## 6. Decision rule (pre-registered)

Per discipline, against the thresholds fixed before running:

- **KEEP** — primary margin ≥ the pre-registered minimum AND null-fixture FP below its threshold.
- **CUT** — a *positive harm signal*: negative point estimate on the primary, OR null-fixture FP at/over threshold (FP **overrides** a positive primary — a discipline that helps real cases but fires on a meaningful share of null cases is net-harmful in a gating pipeline). Update the CHANGELOG status from "unmeasured" to **"measured-{neutral|harmful} on \<model\>/\<effort\>, family X — removed,"** model+effort inline.
- **INCONCLUSIVE** (the expected default at this N) — anything between. Rerun at larger N only if cheap; otherwise downgrade the discipline to opt-in rather than removing it. **Neutral is never CUT.**

This is the mechanism that makes "effect unmeasured" temporary and self-correcting — and, by the asymmetry + FP-override rules, makes it safe.

---

## 7. Minimal first pass (this is the whole commitment; the rest is optional)

Run the **two** disciplines with crisp oracles, near-blind scoring, and asymmetric value:

1. **M1 — render-leak escape rate** (10-15 render fixtures + 5 null). Build + **validate** the render-leak scorer (§4.3); it's the one reused oracle worth automating.
2. **M4 — UNTESTABLE fabrication rate** (10 unreachable + 5 reachable null). **Hand-scored** — do not build a classifier for a one-state prompt change.

Pre-register both thresholds, run ~30-40 paired runs (one focused session), report direction + raw tallies + FP, decide KEEP/CUT/INCONCLUSIVE. **Stop there** unless a result motivates more. The M2/M3/M5/M6/M7 corpus-and-scorer build is explicitly *not* committed by this document — it is gated on the first pass showing the approach pays for itself. Shipping only §7 keeps the measurement lighter than the thing measured; expanding the full §3/§4 program without that evidence would re-introduce the accretion the whole exercise exists to prevent.
